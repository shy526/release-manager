# release-manager
- 着手解决老项目发布困难的自动化脚本


## dubbo项目一般发布流程

1. 登录dubbo网页管理客户端
2. 禁用dubbo相关服务
3. 上传jar或war
4. 备份原来jar或war

--------------------

5. 清理原代码,启动新代码

---------------------

6. 等待服务完全启用

----------------------
7. 启用dubbo相关服务
----------------------
8. 决定是否回滚
9. 清理原代码或保留备份
----------------------
10. 全发布

> 1,2,3,4 java 自动化
>>5 上传shell脚本运行
>>> 6,7 手动确认,全发布时比对样机服务数
>>>> 8,9 手动确认
>>>>> 10 全自动 执行 1234567 跳过9,10
